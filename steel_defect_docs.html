01 / OVERVIEW
What Does This Project Do?

Understand the project in plain simple language — no technical knowledge needed.

Simple Explanation: This project is an AI system that looks at a photo of a steel plate and tells you exactly what kind of surface defect it has — and shows you WHERE the defect is using a colour heatmap.

The Problem It Solves

😴
Human Fatigue
Human inspectors get tired checking thousands of plates daily. Fatigue causes missed defects that reach customers.
🐢
Slow Speed
Manual inspection limits how many plates can be checked per hour. AI checks each image in under 100 milliseconds.
❓
No Explanation
Most AI models give answers without showing why. Our Grad-CAM shows exactly which part of the image triggered the decision.
💸
High Cost
Defective products that pass inspection cause costly recalls and safety hazards. AI provides consistent 98.3% accuracy.

File Purpose Cards

main.py The Boss / Conductor
▾

This is the entry point — the first file that runs. Think of it as an orchestra conductor. It doesn't play any instrument itself but tells every other file when to do what.

When you type python main.py, it: creates folders → loads data → builds model → trains → saves results. It calls functions from all other files in the correct order.

predict.py The Demo / Inference File
▾

After training is complete, you use this file to test new images. It loads the saved model and runs it on any image you give it.

Run it like: python predict.py --image "path/to/image.jpg". It outputs the predicted class, confidence percentage, and saves the Grad-CAM heatmap images.

src/config.py The Settings Dashboard
▾

All important numbers and paths are stored here in one place. IMAGE_SIZE = (224, 224), BATCH_SIZE = 32, NUM_CLASSES = 6, folder paths, etc.

Why? So you never have to search the entire codebase to change one setting. Change it once here, it reflects everywhere.

src/data.py The Data Preparation Factory
▾

Finds all images on disk, builds a table of (filepath → label), splits validation into val+test, and wraps everything into generators — smart objects that load images in batches during training so RAM doesn't overflow.

Also applies data augmentation — rotation, flipping, zoom, brightness changes — to training images to create variety.

src/model.py The Brain Blueprint
▾

Defines the entire neural network structure. Downloads ResNet50 pretrained weights, freezes them, adds the custom classification head (Dense layers), and compiles with Adam optimizer and categorical crossentropy loss.

This file answers: "What does the neural network look like?"

src/gradcam.py The Explainability Engine
▾

The most unique file in the project. Uses mathematical gradients to figure out which parts of the image most influenced the model's decision, then renders those as a red-yellow-green heatmap overlay on the original image.

This is what makes the system trustworthy — you can see the AI's "reasoning" visually.

How the Code Executes

Training Pipeline (main.py)

1
Folders Created
ensure_directories() — Creates models/ and outputs/ folders if they don't exist yet. Also sets random seed to 42 so results are reproducible.
2

3
Validation Split into Val + Test
split_validation_test() — Takes the validation DataFrame and splits it 50/50 using stratified splitting (ensures each class gets equal representation in both halves). No files are moved on disk.
4
Generators Created
create_generators() — Wraps the DataFrames into Keras generators. Training generator applies augmentation. Val and test generators just normalize. Generators load images in batches of 32 on-demand.
5
Model Built
build_model() — Downloads ResNet50 pretrained weights (first run only), freezes the backbone, adds custom head layers, compiles with Adam + crossentropy. Ready to train.
6
Training Begins
model.fit() — The main training loop. Feeds batches of 32 images, computes predictions, calculates loss, runs backpropagation to update weights. Callbacks monitor and auto-save the best model.
7
Results Saved
plot_training_history() + evaluate_and_save() — Saves accuracy curve, loss curve, confusion matrix, classification report, and metrics JSON to the outputs/ folder.

def build_image_dataframe(images_dir):
    records = []
    for class_name in CLASS_NAMES:          # loop through 6 defect folders
        class_dir = images_dir / class_name   # build path to each folder
        for image_path in class_dir.glob("*"): # find every image file inside
            records.append({
                "filepath": str(image_path), # full path to the image
                "label": class_name          # what class it belongs to
            })
    return pd.DataFrame(records)   # creates a table: filepath | label

# Result is a table like this:

model.py — Building ResNet50

Python
# Load ResNet50 with ImageNet weights — already knows edges, textures, patterns
base_model = ResNet50(
    weights="imagenet",      # pretrained on 1.2M images — don't start from scratch
    include_top=False,      # remove original 1000-class head — we don't need it
    input_shape=(224,224,3) # our images are 224x224 with 3 color channels
)
base_model.trainable = False  # FREEZE — don't change pretrained weights in Stage 1

# Build the full model with our custom head on top
inputs = layers.Input(shape=(224, 224, 3))    # the door images enter through
x = base_model(inputs, training=False)          # pass through all 50 ResNet layers
x = layers.GlobalAveragePooling2D()(x)          # compress 7x7x2048 → just 2048 numbers
x = layers.Dense(256, activation="relu")(x)   # 256 neurons learn defect patterns
x = layers.Dropout(0.4)(x)                     # randomly drop 40% neurons — prevents memorization
outputs = layers.Dense(6, activation="softmax")(x) # 6 outputs, one per defect class

model.compile(
    optimizer=Adam(learning_rate=1e-4),      # Adam: smart, adaptive learning rate
    loss="categorical_crossentropy",           # measures how wrong predictions are
    metrics=["accuracy", Precision(), Recall()] # what to track and report
)

gradcam.py — The Heatmap Magic

Python
# Create special model with 2 outputs: conv layer output + final prediction
grad_model = tf.keras.models.Model(
    [model.inputs],
    [model.get_layer("resnet50").get_layer("conv5_block3_out").output, model.output]
)

# GradientTape records all math operations so we can compute gradients later
with tf.GradientTape() as tape:
    conv_outputs, predictions = grad_model(image_batch)
    target_score = predictions[:, predicted_index]  # score of predicted class

# HOW MUCH does each location in the conv layer affect the prediction?
gradients = tape.gradient(target_score, conv_outputs)

# Average importance across all channels — one importance number per location
pooled_gradients = tf.reduce_mean(gradients, axis=(0, 1, 2))

# Weight the feature maps by their importance → gives us the heatmap
heatmap = conv_outputs @ pooled_gradients[..., tf.newaxis]
heatmap = tf.maximum(heatmap, 0)  # keep only positive influences

# Result: a small heatmap that gets resized and overlaid on the original image
# Red/hot = "this region drove the prediction"
# Blue/cool = "this region was not important"

06 / CONCEPTS
Deep Learning from Scratch

Every concept used in this project explained from zero — with analogies and examples.

🧒
Deep Learning
ANALOGY: Teaching a child to recognize dogs
You don't write rules like "4 legs + fur = dog." You show 1000 dog photos and 1000 non-dog photos. The child learns the pattern. Deep learning works identically — you show it labelled examples and it figures out the rules itself.
🕸️
Neural Network
ANALOGY: Chain of people passing notes
A series of mathematical layers. Each layer receives input, transforms it, passes to the next. Layer 1 detects edges, Layer 2 detects shapes, Layer 3 detects patterns. By the end, complex decisions emerge from simple operations.
🔬
CNN (Conv Net)
ANALOGY: Sliding magnifying glass
A CNN slides a small filter (like a 3×3 window) across the entire image. At each position it asks: "Is there a horizontal edge here? A crack? A dark spot?" Hundreds of filters run simultaneously, each detecting one specific feature.
📊
Softmax
ANALOGY: Voting percentage counter
Converts 6 raw numbers into percentages that add up to 100%. Output: [2%, 1%, 3%, 2%, 1%, 91%] means 91% confident it's class 6 (scratches). The highest percentage wins as the predicted class.
📉
Loss Function
ANALOGY: Exam score gap
Measures how wrong the prediction was. If correct class gets 90% probability → small loss. If correct class gets 5% → huge loss. Training tries to minimize this number over thousands of iterations.
🎛️
Optimizer (Adam)
ANALOGY: Smart study coach
After backpropagation calculates what to fix, Adam decides HOW MUCH to adjust each weight. It gives bigger adjustments to rarely-updated weights and smaller ones to frequently-updated ones. Smarter than simple gradient descent.
🔄
Backpropagation
ANALOGY: Reviewing an exam
After making a wrong prediction, the error travels backwards through the network. Each weight learns how much it contributed to the mistake. Weights that caused the error get adjusted more. This is how the model "learns."
🎲
Dropout
ANALOGY: Studying with covered notes
During training, 40% of neurons are randomly switched off each step. This prevents the model from memorizing training images. It forces every neuron to learn independently. Result: better generalization to new images.

ReLU Activation Function

What it does: output = max(0, x) — Any negative number becomes 0. Positive numbers pass through unchanged.

Why we need it: Without activation functions, stacking 50 layers would just be one giant multiplication — still a straight line. ReLU adds non-linearity, giving the network the ability to learn curved, complex decision boundaries.

Where used: After every Dense layer in our custom head: Dense(256, activation="relu")

ResNet50 Architecture

Why ResNet50, how it works, and what makes it better than other options.

Architecture Diagram

📸 Input 224 × 224 × 3
↓
🏗️ ResNet50 Backbone Frozen in Stage 1
↓
🔗 Skip Connections 16 Residual Blocks
↓
📐 GlobalAveragePooling2D 7×7×2048 → 2048
↓
⚡ Dense 256 + ReLU Custom Head
↓
🎲 Dropout 0.4 40% neurons dropped
↓
🎯 Dense 6 + Softmax Output probabilities

The Skip Connection Innovation

❌
Normal Deep Network Problem
When you stack 50 layers, gradients become tiny during backpropagation — multiplied by small numbers 50 times, they vanish to near-zero. Early layers stop learning. This is the Vanishing Gradient Problem.
✅
ResNet's Solution
Skip connections add a shortcut: Output = Layer(Input) + Input. Gradients always have a highway to travel back through. Even 50 layers deep, every layer gets meaningful gradient signals and keeps learning.

Two-Stage Training Strategy

Stage 1 — Frozen Backbone: ResNet50 weights are locked. Only Dense layers train. Learning rate: 1e-4. Goal: teach the head what combination of ResNet features = each defect class.
Stage 2 — Fine-Tuning: Top ResNet layers are unfrozen. Entire model trains together. Learning rate: 1e-5 (very low, to not destroy pretrained knowledge). Goal: adapt deep features specifically to steel defect textures.
